## 5. UTM-схема и цели

**Шаблон UTM:**
`?utm_source=ИСТОЧНИК&utm_medium=cpc&utm_campaign=КАМПАНИЯ&utm_content=ОБЪЯВЛЕНИЕ&utm_term={keyword}`

**Примеры:**
- Директ-поиск, двигатель: `?utm_source=yandex&utm_medium=cpc&utm_campaign=search_engine&utm_content=group_C&utm_term={keyword}`
- РСЯ, ретаргет: `?utm_source=yandex&utm_medium=cpc&utm_campaign=rsya_retarget`
- VK Ads, ходовая: `?utm_source=vk&utm_medium=cpc&utm_campaign=hodovaya&utm_content=video_dop`

UTM автоматически прикрепляются к каждой заявке (реализовано в `script.js` → `sendLead`).

**Цели Яндекс.Метрики / события VK-пикселя (уже шлются из кода):**
| Событие | Когда срабатывает |
|---|---|
| `lead` | любая отправленная заявка (квиз/CTA/модалка) |
| `quiz_lead` | заявка именно из квиза |
| `quiz_step` | прохождение шага квиза (микроконверсия) |
| `call` | клик по телефону |
| `vk` | клик по ВК |
| `open_modal` | открытие формы записи |

Настройте эти цели в Метрике и конверсии в VK Ads — и можно оптимизировать кампании по `lead`.

## 7. Чек-лист запуска
1. Зарегистрировать домен и опубликовать лендинг (HTTPS).
2. В `script.js` → `CONFIG`: вписать `metrikaId`, `vkPixelId`, `vkGroupId`, при наличии — `leadEndpoint` (Formspree/Getform/CRM).
3. Включить «Сообщения сообщества» в VK для приёма заявок через vk.me/rpv51.
4. Создать счётчик Метрики + цели; пиксель VK + конверсии.
5. Подготовить политику обработки ПДн (ссылка под формами) — требование модерации.
6. Маркировка рекламы (ОРД, erid) — при необходимости через рекламные кабинеты.
7. Запустить кампании с UTM, отслеживать `lead`/`quiz_lead`, докручивать ставки по группам.

> **Совет по модерации:** избегайте «лучший/№1»; «бесплатно» подтверждается условием на лендинге (это ок). Гарантию формулируйте как «гарантия на работы» (есть на лендинге).
